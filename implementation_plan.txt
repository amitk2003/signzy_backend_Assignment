INTELLIGENT VENDOR ROUTING PLATFORM — IMPLEMENTATION PLAN
==========================================================

PROBLEM SUMMARY
---------------
Build a backend system that acts as a smart middleman between clients and 
multiple third-party vendors. The system picks the best vendor for each request 
based on routing rules, tracks how vendors are performing, and switches vendors 
automatically when something goes wrong.

Example flow: Client -> /route (PAN_VERIFICATION) -> System picks VendorA -> Gets response -> Returns standardized response

The client never knows which vendor handled the request.


TECH STACK
----------
- Runtime:    Node.js
- Framework:  Express.js
- Database:   In-memory (Maps/Arrays) — no external DB needed
- Language:   JavaScript (ES6)

ROUTING STRATEGIES (5 out of 8)
-------------------------------
1. Priority-based   — Picks vendor with lowest priority number (highest priority)
2. Weighted          — Distributes traffic based on weight % (e.g., 70/30)
3. Lowest Latency    — Picks vendor with lowest average response time
4. Lowest Cost       — Picks vendor with cheapest cost per request
5. Round Robin       — Cycles through vendors in order


FAILOVER LOGIC
--------------
When primary vendor fails, system automatically tries next vendor.
Reasons for skipping a vendor:
  - Vendor is marked unhealthy (high error rate or latency)
  - Rate limit exceeded
  - Vendor doesn't support required feature
  - Vendor call timed out or errored

Each skip reason is logged in routing decision logs.


EDGE CASES HANDLED
------------------
- No vendors registered for a capability -> clear error
- All vendors down -> error after exhausting all options
- Rate limit exceeded on all vendors -> error with retry hint
- Invalid strategy name -> falls back to priority-based
- Missing required fields -> validation errors
- Vendor registered twice -> updates existing
- Metrics for zero calls -> returns zeros (not NaN)


MANDATORY APIs
--------------
POST /vendors           — Register a new vendor
GET  /vendors           — List all vendors
POST /route             — Route a request to best vendor
GET  /vendor-metrics    — Get vendor performance metrics
GET  /routing-logs      — Get routing decision history
GET  /health            — Health check
